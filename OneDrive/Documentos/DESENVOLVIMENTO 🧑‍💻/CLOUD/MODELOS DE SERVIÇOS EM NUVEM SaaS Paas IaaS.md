

1. **IaaS (Infraestrutura como Serviço):**
    
    - **Cenário:** Imagine que você precisa hospedar um servidor web.
    - **Aplicação:** Em vez de comprar e manter o hardware físico, você aloca uma máquina virtual na nuvem (Azure, AWS, etc.). O provedor de nuvem cuida do hardware (disco, memória, etc.), e você é responsável por instalar e manter o sistema operacional, as aplicações e os patches.
    - **Verificação:** Você está gerenciando o sistema operacional e as aplicações, mas não se preocupa com a infraestrutura física subjacente.




2. **PaaS (Plataforma como Serviço):**
    
    - **Cenário:** Você precisa de um banco de dados para sua aplicação.
    - **Aplicação:** Em vez de configurar um servidor, instalar e manter o software do banco de dados, você usa um serviço de banco de dados na nuvem (como Azure SQL Database). O provedor cuida do sistema operacional, do software do banco de dados e das atualizações.
    - **Verificação:** Você está usando o banco de dados como uma plataforma, sem se preocupar com a manutenção do software ou do sistema operacional.



3. **SaaS (Software como Serviço):**
    
    - **Cenário:** Você precisa de um software de CRM (Customer Relationship Management).
    - **Aplicação:** Em vez de comprar, instalar e manter um software de CRM, você usa um serviço baseado na nuvem, como o Salesforce ou o Microsoft Dynamics 365. O provedor cuida de tudo, desde a infraestrutura até o software e as atualizações.
    - **Verificação:** Você está apenas usando o software, sem se preocupar com nenhum aspecto da infraestrutura ou da manutenção do software.
