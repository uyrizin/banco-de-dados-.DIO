

# **Attention Mechanism**

Técnica usada por modelos de linguagem, como o Transformer, que permite que o modelo foque em partes específicas do texto de entrada ao gerar a resposta.

# **Chunks**

Segmentos ou unidades de maior nível de significado em um texto, geralmente consistindo de palavras ou frases, que ajudam a entender a estrutura semântica do texto.

# **Contexto Longo**

Refere-se à capacidade de um modelo de linguagem processar informações de um prompt maior, mantendo a coerência e sem perder informações importantes.

**Contextualização**

O processo de entender as palavras dentro do contexto em que aparecem.

# **Embeddings**

Representações vetoriais de palavras ou frases, capturando seu significado semântico e usadas para comparar palavras ou expressões em termos de semelhança.

# **Feedforward Network**

Tipo de rede neural em que as informações seguem uma direção única, sem ciclos, do início ao fim da rede, com camadas de processamento intermediárias.

# **Few-shot Learning**

Tipo de aprendizado em que o modelo aprende com muito poucos exemplos ou dados, com base em conhecimento pré-existente.

# **Fine-tuning**

Ajuste de um modelo de linguagem pré-treinado com dados específicos para tarefas ou contextos, melhorando a performance do modelo.

# **Guardrails**
# 
Limitações ou diretrizes estabelecidas para garantir que o modelo de linguagem gere respostas dentro de limites aceitáveis e seguras, evitando conteúdos indesejados.

# **Inferência**

Processo de gerar uma saída usando os dados treinados, como uma resposta a um prompt, após o treinamento do modelo.

# **Janela de Contexto**

Número de tokens que o modelo consegue processar de uma vez; quando o limite é atingido, o modelo pode "esquecer" informações mais antigas.

# **Modelo de Linguagem**

Tipo de modelo de IA treinado para entender e gerar texto, como GPT, BERT e T5.

# **N-grama**

Sequência de n tokens consecutivos, usados para capturar relações contextuais entre palavras. Ex: unigramas, bigramas, trigramas.

# **Overfitting**

Quando um modelo aprende demais sobre os dados de treinamento, perdendo a capacidade de generalizar para novos dados.

# **Parâmetros de Aprendizagem**

Elementos configuráveis de um modelo de linguagem, como taxa de aprendizado, que afetam o treinamento.

# **Parâmetros do Modelo**

Variáveis internas ajustadas durante o treinamento de um modelo de linguagem, influenciando sua capacidade de aprender complexidades.

# **Parsing**

Análise estrutural do texto para entender sua sintaxe e relações gramaticais, ajudando na identificação da estrutura da frase.

# **Prompt Engineering**

Processo de projetar e otimizar prompts para obter respostas mais eficazes de um modelo de linguagem.

# **Prompts**

Texto ou entrada fornecida ao modelo de linguagem para gerar uma resposta.

**Reconhecimento de Entidade Nomeada (NER)**
# 
Técnica para identificar e classificar partes específicas do texto, como nomes de pessoas, locais e datas.

**RAG (Retrieval-Augmented Generation)**

Técnica que combina a recuperação de informações relevantes (de uma base de dados ou documentos) com a geração de respostas, melhorando a precisão e relevância.

# **Stopwords**

Palavras comuns, como "a", "o", "de", que são descartadas no processamento por não terem valor semântico relevante na maioria dos contextos.

# **Stemming**

Processo de reduzir palavras às suas raízes, ajudando o modelo a compreender palavras com variações morfológicas.

# **Token**

Unidade básica de processamento de texto, podendo ser uma palavra, parte de uma palavra ou até um caractere.

## **Tokenização**

Processo de dividir um texto em tokens, que são unidades de linguagem que os modelos de linguagem podem processar.

# **Underfitting**

Quando um modelo não aprende o suficiente sobre os dados de treinamento, resultando em um desempenho ruim.

# **Zero-shot Learning**

Técnica que permite ao modelo realizar tarefas sem ter sido explicitamente treinado para elas, utilizando seu conhecimento pré-existente.
